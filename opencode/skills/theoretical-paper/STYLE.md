# STYLE – Theoretical Paper Conventions

This file extends the shared conventions in `/home/random_user/.config/opencode/STYLE.md`.
Read the root `/home/random_user/.config/opencode/STYLE.md` first; everything there applies (72-char wrap,
heading line breaks, OpenXML page breaks, custom styles, hamburger
paragraphs, etc.).

## 1. Voice and Framing

- Theoretical papers use the same confident, matter-of-fact academic
  voice as empirical papers (see `/home/random_user/.config/opencode/STYLE.md` §3).
- Lean more heavily on **examples, metaphors, and thought experiments**
  to ground abstract arguments. A good running example introduced early
  should reappear throughout to anchor the reader.

## 2. Argument Structure

- Organize the body into **arguments, not authors**: each major section
  advances one key claim, supported by sub-claims, evidence, and
  engagement with counter-arguments.
- When challenging existing norms, demonstrate understanding of the
  current norm before arguing for change. Show continuity where possible
  to reduce perceived upheaval.
