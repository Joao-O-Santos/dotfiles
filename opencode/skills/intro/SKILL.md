# Skill: intro

#style-core

## Inputs

- `paper_type`: empirical | theoretical
- Existing outline, notes, or draft (if revising)

## Structure

### Empirical Papers

1. **Hook** – concrete, provocative, accessible to a lay reader
   #intro-hook-empirical

2. **Background** – conceptual/historical context, organized by theme

3. **Gap + Contribution** – what's missing, what this paper adds

### Theoretical Papers

1. **Hook** – concrete example/metaphor or quote
   #intro-hook-theoretical

2. **Background** – how the field currently thinks about this issue

3. **Thesis** – explicit statement of the core claim

## Common Mistakes

- Don't open with vague disciplinary overview; start concrete
- Don't bury the main claim; make thesis explicit by end
- Don't organize author-by-author; organize by theme/claim

## Workflow

1. If drafting: produce topic-sentence outline first
2. If revising: reverse-outline existing draft, propose structural changes
3. Expand to prose, using `<!-- TODO: cite -->` for missing references
